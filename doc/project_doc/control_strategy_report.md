# Black Pearl v1.1 控制闭环策略汇报 README

> 本文档用于说明在当前硬件条件下，船体控制应该优先闭什么环、哪些闭环暂时不适合做，以及后续升级到航向保持和导航控制需要补齐哪些条件。

---

## 1. 当前硬件条件

当前工程已经具备以下执行器和传感器：

| 类型 | 当前硬件/模块 | 能提供什么 | 当前限制 |
|------|---------------|------------|----------|
| 执行器 | 左右双电机 PWM | 通过左右差速控制前进、后退、转向 | 只有 PWM 输出，没有电机转速反馈 |
| IMU | QMI8658 | 三轴加速度、三轴陀螺仪 | QMI8658 曾出现 I2C 不稳定；陀螺有零偏，需要静止学习 |
| 磁力计 | QMC6309 | 地磁航向参考 | 未做硬铁/软铁校准，容易受电机、船体金属和线缆电流影响 |
| GPS | UART2 NMEA | 经纬度、地速、航迹角 `course` | 静止、低速、原地转向时航迹角不可靠 |
| 无线 | LT8920 | 遥控指令和数据链路 | 不是姿态或速度反馈源 |
| 控制算法 | PID 定点模块 | 可复用做角速度环、航向环、速度环 | 速度环缺少真实速度反馈，暂不适合直接闭电机速度 |

关键结论：

- 当前没有电机编码器、桨轴转速、霍尔测速、水速计或电流传感器，所以**不能做真正的电机转速闭环或推力闭环**。
- 当前最可靠的快速反馈量是 **QMI8658 陀螺 Z 轴角速度**，所以第一阶段最适合做 **偏航角速度闭环**。
- 磁力计 yaw 和 GPS course 可以作为低频航向参考，但在当前未校准阶段，不适合直接高增益闭环。

---

## 2. 不建议直接做的闭环

### 2.1 不建议做电机速度闭环

当前 `Motor_SetBothSpeed(left, right)` 里的 `speed=-1000~+1000` 本质是 PWM 命令，不是实测电机转速。

没有编码器时：

```text
目标电机速度 -> PWM -> 电机实际转速
```

中间缺少反馈，无法知道电机实际有没有达到目标。水草、负载、电池电压、左右桨效率差都会让同一个 PWM 对应不同推力。

因此当前只能做：

```text
油门 base_pwm：开环
左右差速 delta_pwm：由姿态/角速度闭环修正
```

### 2.2 不建议用 roll/pitch 闭航向

roll/pitch 是船体横滚和俯仰，主要用于判断姿态是否异常、是否倾斜过大。普通双电机差速船没有舵面或姿态执行器，不能通过 roll/pitch 直接控制航向。

roll/pitch 当前适合作为：

- 姿态状态监测
- 异常保护条件
- AHRS 质量判断辅助

不适合作为航向闭环主反馈。

### 2.3 不建议直接用未校准磁力计 yaw 高增益闭环

当前 QMC6309 已经能出数，但还没有完成：

- 硬铁 offset 校准
- 软铁比例校准
- 电机通电后的磁干扰评估
- 倾斜补偿 yaw

如果直接用磁力计 yaw 高增益控制电机差速，可能出现：

```text
电机电流改变磁场 -> yaw 被拉偏 -> 控制器继续修正 -> 电机差速更大 -> 磁干扰更强
```

这会形成错误反馈。当前磁力计只建议作为低频参考或调试观测，不建议直接作为主闭环源。

### 2.4 不建议低速用 GPS course 闭航向

GPS 的 `course_deg_x100` 是航迹角，表示地面运动方向，不等于船头朝向。

低速或原地转向时：

- GPS 位置噪声会放大成 course 抖动
- 船头转了，但位置没明显移动，GPS course 不跟随
- 水流、漂移会让航迹角和船头方向不同

因此 GPS course 只适合在船已经有明显前进速度时作为巡航方向参考，不适合静止和低速掉头闭环。

## 3. 当前最合适的闭环结构

当前推荐采用两层结构：

```text
外层：航向/转向意图
  -> 生成目标偏航角速度 target_yaw_rate

内层：陀螺 Z 轴角速度闭环
  -> 生成左右电机差速 delta_pwm
```

第一阶段只先做内层，也就是 **Z 轴角速度闭环**。

### 3.1 第一阶段：偏航角速度闭环

目标：

- 遥控打方向时，船按可控角速度转向
- 遥控不打方向时，抑制船自己偏航和自旋
- 不依赖磁力计和 GPS，先把电机差速方向、陀螺符号和 PID 跑通

控制量：

```text
measured_yaw_rate = AHRS_State.gyro_z_dps100
target_yaw_rate   = 遥控转向量换算得到，单位 deg/s * 100
```

控制流程：

```text
base_pwm = 油门命令
rate_error = target_yaw_rate - measured_yaw_rate
delta_pwm = PID_Rate(rate_error)

left_pwm  = base_pwm - delta_pwm
right_pwm = base_pwm + delta_pwm
```

左右加减号需要实船确认。如果发现打右转却左转，只交换 `delta_pwm` 符号即可。

建议门控：

- `QMI8658` 初始化失败时，不启用闭环。
- `flags` 缺少 `GYRO_BIAS_READY` 时，不启用自动航向保持。
- `DT_CLAMPED` 出现时，暂停 PID 积分或降低控制输出。
- 切换控制模式、解锁、上锁时调用 `PID_Reset()`。

### 3.2 第二阶段：短时航向保持

在角速度环稳定后，再加外层航向保持。

适用场景：

- 遥控油门给定，方向杆回中
- 希望船短时间保持当前船头方向

控制流程：

```text
方向杆回中瞬间：
    target_heading = current_yaw_or_yr

运行中：
    heading_error = wrap(target_heading - current_heading)
    target_yaw_rate = clamp(Kp_heading * heading_error)
    delta_pwm = PID_Rate(target_yaw_rate - gyro_z)
```

这里推荐外环先用 P 控制，不要一开始就加 I：

- 外环 I 容易在磁力计/GPS 不可靠时积累错误
- 船体水动力响应慢，外环过强会导致左右摆动
- 内层角速度环已经负责快速动态，外环只负责慢慢拉回方向

当前 `yr` 是调试相对航向，可以用于验证 90 度旋转和短时航向保持。但在磁力计未校准前，不建议把 `yr` 当成长期绝对航向。

### 3.3 第三阶段：地磁低频修正

当 QMC6309 完成校准后，可以把地磁 yaw 作为低频参考。

推荐用法不是直接：

```text
mag_yaw_error -> 电机差速
```

而是：

```text
mag_yaw -> 慢修正 AHRS yaw / heading reference
AHRS yaw -> 航向外环
gyro_z -> 角速度内环
```

地磁参与控制前至少要满足：

- `MAG_VALID=1`
- 磁场模长在合理窗口内
- 电机不同 PWM 下 yaw 不出现明显跳变
- 已完成硬铁 offset 校准
- 已确认磁力计轴向和 IMU 轴向一致

### 3.4 第四阶段：GPS 航迹辅助

GPS course 适合做直线巡航或航点导航的低频参考。

建议使用条件：

```text
GPS fix 有效
speed_kmh_x100 > 150~200
course 数据连续更新
```

含义是船速至少约 `1.5~2.0 km/h` 后，再相信 GPS course。低于这个速度时，GPS course 应直接禁用。

GPS course 推荐放在最外层：

```text
目标航点 -> 期望航迹角
GPS course / GPS 位置 -> 航迹误差
航迹误差 -> target_heading
target_heading -> target_yaw_rate
gyro_z -> delta_pwm
```

它不替代 IMU 角速度环。GPS 太慢，不能直接拿来修电机差速。

## 4. 推荐控制模式表

| 阶段 | 模式 | 反馈源 | 输出 | 是否建议当前实现 |
|------|------|--------|------|------------------|
| 0 | 纯手动开环 | 无 | `left/right PWM` | 已可做 |
| 1 | 角速度阻尼/转向闭环 | `gyro_z` | `delta_pwm` | 最优先 |
| 2 | 短时航向保持 | `yr` 或 AHRS yaw + `gyro_z` | `target_yaw_rate` + `delta_pwm` | 角速度环稳定后做 |
| 3 | 地磁航向保持 | 校准后 mag yaw + `gyro_z` | `target_yaw_rate` + `delta_pwm` | 校准后做 |
| 4 | GPS 直线巡航 | GPS course + `gyro_z` | `target_heading` + `delta_pwm` | 有速度后做 |
| 5 | 航点导航 | GPS 位置 + heading/rate 双环 | 左右电机命令 | 后续升级 |

当前最合适的汇报结论：

```text
本阶段不直接做绝对 yaw 闭环。
先以 QMI8658 陀螺 Z 轴建立偏航角速度内环，实现转向阻尼和可控转向。
在角速度环稳定后，再叠加短时航向保持外环。
磁力计和 GPS 只作为低频航向参考，必须经过质量门控后才能参与控制。
```

## 5. 具体落地方案

### 5.1 控制输入定义

遥控输入可以拆成两个量：

```text
throttle_cmd：前进/后退油门
steer_cmd：左/右转向
```

建议映射：

```text
base_pwm = throttle_cmd
target_yaw_rate = steer_cmd * MAX_YAW_RATE
```

例如：

```text
MAX_YAW_RATE = 90 deg/s
```

则满舵对应目标转向角速度约 `90 deg/s`。实际值要下水调，先从小值开始。

### 5.2 角速度环输出

PID 输出不要一开始给满。建议先限制在电机范围的 20% 到 30%：

```text
delta_pwm_limit = 200~300
```

最终电机命令：

```text
left_pwm  = clamp(base_pwm - delta_pwm, -1000, +1000)
right_pwm = clamp(base_pwm + delta_pwm, -1000, +1000)
```

低速原地转向时，也可以让：

```text
base_pwm = 0
left_pwm  = -delta_pwm
right_pwm = +delta_pwm
```

是否允许原地反转取决于电机和驱动硬件能力，第一次实船测试应限制输出。

### 5.3 航向保持外环

方向杆回中并保持一小段时间后，锁定当前航向：

```text
target_heading = current_heading
```

之后只要方向杆仍回中：

```text
heading_error = wrap(target_heading - current_heading)
target_yaw_rate = clamp(Kp_heading * heading_error, -rate_limit, +rate_limit)
```

推荐先用：

```text
rate_limit = 30~45 deg/s
heading_deadband = 2~3 deg
```

避免船一直因为小误差左右摆。

## 6. 调试顺序

建议按下面顺序做，不要跳级：

1. **确认 IMU 稳定**
   - 上电后 `QMI8658 WHO_AM_I=0x05`
   - AHRS 输出 `f=1F`
   - 静止时 `g.z` 接近 0

2. **确认陀螺 Z 轴符号**
   - 手动顺时针旋转船体
   - 记录 `g.z` 是正还是负
   - 固定“顺时针为正”或“逆时针为正”的工程约定

3. **确认电机差速符号**
   - 小油门测试 `left_pwm > right_pwm` 时船向哪边转
   - 如果和控制预期相反，只反 `delta_pwm` 符号，不要乱改 AHRS 轴

4. **只调角速度 P**
   - `Ki=0`
   - `Kd=0`
   - 慢慢加 `Kp`
   - 目标是转向跟手但不左右摆

5. **再加少量 D 或滤波**
   - 如果转向超调明显，再考虑 D
   - 如果噪声大，优先滤波和限幅，不要直接加大 D

6. **最后加航向保持外环**
   - 外环先只用 P
   - 外环输出的是 `target_yaw_rate`
   - 内环继续闭 `gyro_z`

## 7. 控制质量门控

闭环控制必须有降级逻辑：

| 条件 | 动作 |
|------|------|
| IMU 初始化失败 | 禁用自动闭环，保留手动或停车 |
| `GYRO_BIAS_READY=0` | 不进入航向保持，只允许手动开环或低增益阻尼 |
| `DT_CLAMPED=1` | 暂停 PID 积分，限制 `delta_pwm` |
| `MAG_VALID=0` | 禁用地磁航向修正，保留陀螺角速度环 |
| GPS 未定位 | 禁用 GPS course 控制 |
| GPS 速度过低 | 禁用 GPS course 控制 |
| 磁场模长异常 | 暂停磁力计 yaw 修正 |
| 模式切换/失联/急停 | `PID_Reset()` 并逐步收输出 |

这部分比算法形式更重要。船在水上受干扰大，错误传感器参与闭环比没有闭环更危险。

## 8. 阶段验收标准

第一阶段角速度闭环通过标准：

1. 静止放平后 AHRS 进入 `f=1F`。
2. 船体不转时，`gyro_z` 接近 0。
3. 手动给定固定 `target_yaw_rate`，船能稳定向同一方向转。
4. 方向杆回中后，船的自旋趋势明显减小。
5. `delta_pwm` 不长期打满，左右电机没有高频抖动。

第二阶段短时航向保持通过标准：

1. 方向杆回中后，系统能锁定当前 `yr/yaw`。
2. 短时间内受到轻微扰动后，船能回到接近原航向。
3. 航向误差变小时，`target_yaw_rate` 和 `delta_pwm` 能随之减小。
4. 失去 IMU ready、`DT_CLAMPED` 或模式切换时，PID 能复位并降级。

第三阶段地磁/GPS 参与控制通过标准：

1. 磁力计完成校准后，旋转一圈 yaw 没有明显畸变。
2. 电机不同 PWM 下，地磁 yaw 不出现不可接受的跳变。
3. GPS 只有在有效定位且速度足够时才参与 course 修正。
4. 地磁和 GPS 只做低频修正，不直接覆盖陀螺角速度内环。

## 9. 对外汇报口径

建议汇报时这样说：

```text
当前硬件不具备电机转速反馈，因此推进器本体不做速度闭环，油门仍为开环 PWM。
当前最可靠的实时反馈是 IMU 陀螺 Z 轴，因此控制闭环优先落在偏航角速度内环。
航向保持采用级联结构：航向误差外环生成目标偏航角速度，陀螺 Z 轴内环生成左右电机差速。
磁力计和 GPS 不作为第一阶段高增益闭环输入，只作为后续低频航向修正源，并通过 flags、磁场模长、GPS 速度和定位质量做门控。
```

这套方案符合当前硬件条件，也给后续升级留下路径：先稳定转向，再做短时航向保持，再接地磁和 GPS，最后做航点导航。
