# QMC6309 地磁计驱动文档

本文档记录 QMC6309 3轴地磁传感器的驱动关键信息，供后续维护参考。

---

## 1. 硬件信息

| 项目 | 说明 |
|------|------|
| MCU | STC32G |
| 总线 | STC32G 硬件 I2C |
| SCL 引脚 | P1.5 |
| SDA 引脚 | P1.4 |
| 系统时钟 | Fosc = 24MHz |
| I2C 地址 | 0x7C（SA=浮空）/ 0x0C（SA=地） |

**注意**：STC32G 硬件 I2C 引脚是固定的（P1.4/P1.5），不能随意更换。

---

## 2. I2C 地址

| 引脚 SA 状态 | 7位地址 | 写地址(tx) | 读地址(rx) |
|--------------|--------|-----------|-----------|
| 浮空/高      | 0x7C   | 0xF8      | 0xF9      |
| 低（接地）   | 0x0C   | 0x18      | 0x19      |

本项目使用 0x7C 地址（SA 浮空/高电平）。

驱动探测顺序：先试 0x7C，失败则回退到 0x0C。

---

## 3. I2C 速度配置

**公式：**
```
I2C速度 = Fosc / 2 / (Speed * 2 + 4)
```

当前配置 SPEED=63，Fosc=24MHz：
```
24MHz / 2 / (63*2+4) ≈ 92.3kHz
```

| SPEED | I2C 速度（@Fosc=24MHz） |
|-------|-------------------------|
| 63    | ~92kHz（当前配置）       |
| 31    | ~185kHz                |
| 20    | ~286kHz                |
| 10    | ~545kHz                |

SPEED 范围 0~63，**值越小速度越快**。

## 4. 关键寄存器

### 4.1 标识寄存器

| 地址 | 名称      | 默认值 | 说明             |
|------|-----------|--------|------------------|
| 0x00 | CHIP_ID   | 0x90   | 芯片标识，只读    |

### 4.2 控制寄存器

| 地址 | 名称        | 说明                    |
|------|-------------|------------------------|
| 0x0A | CONTROL_1   | 传感器配置             |
| 0x0B | CONTROL_2   | ODR 设置 + 模式控制     |

### 4.3 数据寄存器

| 地址 | 名称          | 说明     |
|------|---------------|----------|
| 0x01~0x02 | DATA_X_LSB/MSB | X 轴数据 |
| 0x03~0x04 | DATA_Y_LSB/MSB | Y 轴数据 |
| 0x05~0x06 | DATA_Z_LSB/MSB | Z 轴数据 |

---

## 5. ODR 配置

### 5.1 输出速率 (CONTROL_2 高4位)

| 值   | ODR   | 说明     |
|------|-------|----------|
| 0x00 | 1Hz   | 默认     |
| 0x10 | 10Hz  | 推荐     |
| 0x20 | 50Hz  |          |
| 0x40 | 100Hz |          |
| 0x60 | 200Hz |          |

### 5.2 当前配置

```
CONTROL_1 = 0x1B   // 传感器配置
CONTROL_2 = 0x00   // ODR=1Hz（默认）
```

**注意**：当前默认 ODR=1Hz，非常低。如需更高输出速率，修改 `QMC6309.h` 中的 `QMC6309_CTRL2_INIT`。

## 6. 初始化流程

### 6.1 推荐的初始化序列

```
1. I2C_RecoverBus()           // 先恢复总线
2. 等待上电稳定（1000ms）
3. 探测 CHIP_ID (0x00)        // 确认芯片存在
   - 如果 0x90：芯片正确
   - 如果不是：尝试备用地址
4. 软复位：
   - 写入 CONTROL_2 = 0x80
   - 等待 5ms
   - 写入 CONTROL_2 = 0x00
   - 等待 10ms
5. 写入配置寄存器：
   - CONTROL_1 = 0x00
   - CONTROL_2 = ODR
   - CONTROL_1 = 0x1B
   - 等待 2ms
6. 读回验证
```

### 6.2 当前实现

```c
void compass_configure(void)
{
    // 软复位
    compass_write_reg(CONTROL_2, 0x80);
    compass_delay_ms(5);
    compass_write_reg(CONTROL_2, 0x00);
    compass_delay_ms(10);

    // 配置顺序很重要
    compass_write_reg(CONTROL_1, 0x00);
    compass_write_reg(CONTROL_2, QMC6309_CTRL2_INIT);
    compass_write_reg(CONTROL_1, QMC6309_CTRL1_INIT);
    compass_delay_ms(2);

    // 读回验证
    ctrl1 = compass_read_reg(CONTROL_1);
    ctrl2 = compass_read_reg(CONTROL_2);
}
```

### 6.3 为什么上电等待 1000ms？

QMC6309 上电后需要约 500ms~1000ms 才能稳定输出有效数据。当前使用 1000ms 作为安全余量。

## 7. 读取数据注意事项

### 7.1 多字节连续读取

QMC6309 支持 I2C 自动地址递增，可以一次读取 6 字节 XYZ 数据：

```
START -> 写地址(0xF8) -> ACK
      -> 写寄存器地址(0x01) -> ACK
      -> START -> 读地址(0xF9) -> ACK
      -> 读数据(0x01~0x06) x6 -> NACK
      -> STOP
```

### 7.2 数据格式

QMC6308 数据为 16 位有符号整数，低字节在前（L endian）：

```c
xyz[0] = (int16)(((u16)raw[1] << 8) | raw[0]);  // X
xyz[1] = (int16)(((u16)raw[3] << 8) | raw[2]);  // Y
xyz[2] = (int16)(((u16)raw[5] << 8) | raw[4]);  // Z
```

### 7.3 数据有效性检查

- **帧全0或全0xFF**：说明读取失败或芯片未就绪
- **连续读取数据不变**：可能是 ODR 过低或芯片停滞

---

## 8. 软复位注意事项

### 8.1 软复位流程

```
写入 CONTROL_2 = 0x80
等待 5ms
写入 CONTROL_2 = 0x00
等待 10ms
```

### 8.2 软复位失败的常见原因

1. **I2C 写入失败**：地址发送后收到 NACK
   - 检查 I2C 总线是否被其他设备拉低
2. **CHIP_ID 不是 0x90**：
   - 检查芯片是否正确连接
   - 尝试备用地址 0x0C

---

## 9. I2C 总线注意事项

### 9.1 总线恢复

如果 I2C 总线异常（SDA 或 SCL 被拉低），执行以下恢复步骤：

```
1. 禁用 I2C 模块
2. 配置引脚为 GPIO 输出
3. 发送 9 个时钟脉冲，让从机释放 SDA
4. 发送 STOP 信号
5. 重新配置 I2C
```

### 9.2 总线忙检测

QMC6309 正常工作需要 SDA=1 且 SCL=1。

- **SDA=0, SCL=1**：可能是从机拉低总线，执行恢复
- **SCL=0**：可能是时钟 stretching，执行恢复
- **SDA=1, SCL=1**：总线空闲，可以开始通信

---

## 10. 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| CHIP_ID 读取失败 | I2C 总线异常，地址错误 | 检查总线恢复，SDA是否被拉低 |
| CHIP_ID != 0x90 | 地址错误 | 尝试备用地址 0x0C |
| 数据全0xFF | 芯片未就绪，读取过早 | 等待上电稳定（1000ms） |
| 数据跳变剧烈 | 附近有磁场干扰 | 检查附近是否有磁铁、电机 |
| 航向角不准 | 未校准 | 需要进行硬铁/软铁校准 |
| 初始化超时 | 总线持续被拉低 | 检查 QMI8658 等其他 I2C 设备是否冲突 |

## 12. 优化建议

### 12.1 当前问题

| 问题 | 说明 |
|------|------|
| ODR 默认 1Hz 太低 | 无法满足实时应用需求 |
| 无上电就绪检测 | 只是等待固定时间，不检测实际状态 |
| 无运行时故障恢复 | 数据读取失败后没有自动恢复逻辑 |
| 无温度补偿 | 磁场数据会随温度漂移 |
| 无校准功能 | 无法修正硬铁/软铁误差 |

### 12.2 优化方向

#### 建议 1：提高默认 ODR

```c
// QMC6309.h
#define QMC6309_CTRL2_INIT  QMC6309_ODR_50HZ  // 默认 50Hz
```

#### 建议 2：增加上电就绪检测

```c
static u8 compass_wait_ready(u16 timeout_ms) {
    u16 ms;
    u8 id;
    for(ms = 0; ms < timeout_ms; ms++) {
        id = compass_read_id();
        if(id == QMC6309_CHIP_ID_VALUE) {
            return SUCCESS;
        }
        compass_delay_ms(1);
    }
    return FAIL;
}
```

#### 建议 3：增加运行时故障恢复

```c
u8 compass_read_xyz_safe(int16 xyz[3]) {
    compass_read_xyz(xyz);
    
    // 检查数据有效性
    if((xyz[0] == 0 && xyz[1] == 0 && xyz[2] == 0) ||
       (xyz[0] == -1 && xyz[1] == -1 && xyz[2] == -1)) {
        // 数据无效，尝试重新配置
        if(SUCCESS == compass_configure()) {
            compass_read_xyz(xyz);
            return SUCCESS;
        }
        return FAIL;
    }
    return SUCCESS;
}
```

#### 建议 4：增加校准接口

```c
typedef struct {
    float offset_x, offset_y, offset_z;
    float scale_x, scale_y, scale_z;
    float rotation[3][3];
} QMC6309_Calibration;

void compass_apply_calibration(int16 raw[3], float calibrated[3]);
u8 compass_save_calibration(void);
u8 compass_load_calibration(void);
```

#### 建议 5：增加数据就绪检测

QMC6309 没有 STATUS 寄存器，但可以通过以下方式检测数据就绪：

1. 定时轮询：固定间隔读取
2. 变化检测：检测数据是否发生变化
3. 使用 DRDY 中断（如果有的话）
