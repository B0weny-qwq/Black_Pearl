# QMI8658 IMU 驱动文档

本文档记录 QMI8658 6轴惯性测量单元的驱动关键信息，供后续维护参考。

---

## 1. 硬件信息

| 项目 | 说明 |
|------|------|
| MCU | STC32G |
| 总线 | STC32G 硬件 I2C |
| SCL 引脚 | P1.5 |
| SDA 引脚 | P1.4 |
| 系统时钟 | Fosc = 24MHz |
| I2C 地址 | 0x6B（SA0 浮空/高）/ 0x6A（SA0 低） |

**注意**：STC32G 硬件 I2C 引脚是固定的（P1.4/P1.5），不能随意更换。

---

## 2. I2C 地址

| 引脚 SA0 状态 | 7位地址 | 写地址(tx) | 读地址(rx) |
|---------------|--------|-----------|-----------|
| 浮空/高       | 0x6B   | 0xD6      | 0xD7      |
| 低           | 0x6A   | 0xD4      | 0xD5      |

本项目使用 0x6B 地址（SA0 浮空/高电平）。

驱动探测顺序：先试 0x6B，失败则回退到 0x6A。

---

## 3. I2C 速度配置

**公式：**
```
I2C速度 = Fosc / 2 / (Speed * 2 + 4)
```

当前配置 SPEED=63，Fosc=24MHz：
```
24MHz / 2 / (63*2+4) ≈ 92.3kHz
```

| SPEED | I2C 速度（@Fosc=24MHz） |
|-------|-------------------------|
| 63    | ~92kHz（当前配置）       |
| 31    | ~185kHz                |
| 20    | ~286kHz                |
| 10    | ~545kHz                |

SPEED 范围 0~63，**值越小速度越快**。如需降低速度（长线缆抗干扰），增大 SPEED 值。

## 4. 关键寄存器

### 4.1 标识寄存器

| 地址 | 名称           | 默认值 | 说明                        |
|------|----------------|--------|-----------------------------|
| 0x00 | WHO_AM_I       | 0x05   | 芯片标识，只读              |
| 0x01 | REVISION_ID    | 0x7C   | 版本号，只读                |
| 0x4D | RESET_STATE    | -      | **复位状态，只写0xB0触发**  |

**RESET_STATE (0x4D) 关键行为：**
- 写入 `0xB0` 到 `RESET` 寄存器(0x60)后约 15ms 内，RESET_STATE 应变为 `0x80`
- 如果 RESET_STATE = `0x00`，说明软复位未生效（可能是 I2C 总线问题）
- RESET_STATE 是判断芯片是否就绪的唯一可靠标志

### 4.2 控制寄存器

| 地址 | 名称    | 说明                                    |
|------|---------|----------------------------------------|
| 0x02 | CTRL1   | 数字低通滤波器配置                       |
| 0x03 | CTRL2   | 加速度量程(ACC_RANGE) + 输出速率(ACC_ODR) |
| 0x04 | CTRL3   | 陀螺仪量程(GYRO_RANGE) + 输出速率(GYRO_ODR) |
| 0x05 | CTRL4   | **保留寄存器，无需配置**                |
| 0x06 | CTRL5   | 加速度计配置                            |
| 0x07 | CTRL6   | **保留寄存器，无需配置**                |
| 0x08 | CTRL7   | 传感器使能：Bit7=aEN, Bit3=gEN           |
| 0x09 | CTRL8   | FIFO/ODR 配置（可选）                  |
| 0x0A | CTRL9   | AHB 时钟门控（可选，用于降低功耗）       |

**当前只使用 CTRL1、CTRL2、CTRL3、CTRL5、CTRL7。**

### 4.3 状态寄存器

#### STATUS0 (0x2E)

| Bit | 名称   | 说明               |
|-----|--------|--------------------|
| 0   | aDA    | 加速度数据就绪      |
| 1   | gDA    | 陀螺仪数据就绪      |
| 2   | TempDA | 温度数据就绪        |
| 3   | -      | 保留               |

`STATUS0 = 0x03` 表示加速度 + 陀螺仪数据都就绪。

#### STATUSINT (0x2D)

| Bit | 名称   | 说明                         |
|-----|--------|------------------------------|
| 0   | aSYNCM | 加速度同步完成（syncSmpl 模式） |
| 1   | gSYNCM | 陀螺仪同步完成               |
| 2   | -      | 保留                         |
| 3   | fSyncm | FIFO syncSmpl 触发           |

**注意**：STATUSINT 读取后锁存状态会清除，轮询时需要短暂延时（~5μs）。

### 4.4 数据寄存器

| 地址        | 名称              | 说明                       |
|-------------|-------------------|----------------------------|
| 0x33~0x34  | TEMP_L/H          | 温度（int12，25°C=0）      |
| 0x35~0x3A  | AX_L/H ~ AZ_L/H  | 加速度 xyz                 |
| 0x3B~0x40  | GX_L/H ~ GZ_L/H  | 陀螺仪 xyz                 |

### 4.5 物理量换算

#### 加速度灵敏度（LSB/g）

| 量程 | 灵敏度      |
|------|-------------|
| ±2G  | 16384 LSB/g |
| ±4G  | 8192 LSB/g  |
| ±8G  | 4096 LSB/g  |
| ±16G | 2048 LSB/g  |

#### 陀螺仪灵敏度（LSB/°/s）

| 量程      | 灵敏度        |
|-----------|---------------|
| ±16°/s   | 2048 LSB/°/s |
| ±32°/s   | 1024 LSB/°/s |
| ±64°/s   | 512 LSB/°/s  |
| ±128°/s  | 256 LSB/°/s  |
| ±256°/s  | 128 LSB/°/s  |
| ±512°/s  | 64 LSB/°/s   |
| ±1024°/s | 32 LSB/°/s   |
| ±2048°/s | 16 LSB/°/s   |

**当前配置**：加速度 ±4G，陀螺仪 ±128°/s
- 加速度：8192 LSB/g
- 陀螺仪：256 LSB/°/s

## 6. 初始化流程

### 6.1 推荐的初始化序列

```
1. I2C_RecoverBus()           // 先恢复总线
2. 探测 WHO_AM_I (0x00)       // 确认芯片存在
3. 读取 RESET_STATE (0x4D)    // 检查是否就绪
   - 如果 = 0x80：芯片已就绪，可直接配置
   - 如果 = 0x00：需要软复位
4. 软复位（可选）：
   - 写入 RESET (0x60) = 0xB0
   - 等待 500ms（确保复位完成）
5. 写入控制寄存器（分阶段）：
   - CTRL7 = 0x00（禁用传感器）
   - CTRL1 = 0x40
   - CTRL2 = 量程 + ODR
   - CTRL3 = 量程 + ODR
   - CTRL5 = 0x11
   - CTRL7 = 0x03（启用加速度+陀螺仪）
6. 等待传感器就绪（STATUS0.aDA=1）
7. 读取有效数据帧验证
```

### 6.2 当前配置（默认值）

```
CTRL1 = 0x40   // 数字低通滤波器
CTRL2 = 0x16   // ACC: ±4G, ODR=117Hz
CTRL3 = 0x26   // GYRO: ±128°/s, ODR=117Hz
CTRL5 = 0x11   // 加速度计配置
CTRL7 = 0x03   // aEN=1, gEN=1
```

### 6.3 为什么等待 500ms 软复位？

数据手册要求软复位后等待约 15ms。但代码使用 500ms 是为了确保在所有工况下（时钟偏差、总线延迟）都能稳定。

## 7. 读取数据注意事项

### 7.1 多字节连续读取

QMI8658 支持 I2C 自动地址递增，使用多字节读取可以一次获取所有数据：

```
START -> 写地址(0xD6) -> ACK
      -> 写寄存器地址(0x35) -> ACK
      -> START -> 读地址(0xD7) -> ACK
      -> 读数据(0x35~0x3E) x9 -> NACK
      -> STOP
```

**注意**：多字节读取时，QMI8658 自动递增寄存器地址。

### 7.2 STATUSINT 锁存读取

STATUSINT 寄存器在读取后会清除锁存状态，必须注意：

1. 轮询 STATUSINT 时，每次读取后需要短暂延时（~5μs）
2. 数据读取必须在 STATUSINT 锁存有效时完成
3. 使用 syncSmpl 模式时，读取最后一个字节后自动释放锁

### 7.3 数据有效性检查

- **帧全0或全0xFF**：说明读取失败或芯片未就绪
- **STATUS0 = 0x03**：表示 acc_ready + gyro_ready
- **时间戳递增**：确认帧不是重复数据

## 8. 软复位注意事项

### 8.1 软复位流程

```
写入 RESET (0x60) = 0xB0
等待 500ms（安全余量，数据手册要求约 15ms）
读取 RESET_STATE (0x4D) 应为 0x80
```

### 8.2 软复位失败的常见原因

1. **I2C 写入失败**：地址发送后收到 NACK
   - 检查 I2C 总线是否被其他设备拉低
   - 检查 SDA/SCL 引脚配置是否正确
2. **RESET_STATE 始终为 0x00**：
   - 说明软复位写入没有生效
   - 优先排查 I2C 总线问题
3. **RESET_STATE 变为 0x80 但持续时间很短**：
   - 正常现象，芯片快速完成复位后进入就绪状态

### 8.3 复位后寄存器状态

软复位后，所有配置寄存器恢复默认值：
- CTRL1~CTRL9 = 0x00
- FIFO 相关寄存器 = 0x00
- 只有 WHO_AM_I 和 REVISION_ID 保持不变

## 9. I2C 总线注意事项

### 9.1 总线恢复

如果 I2C 总线异常（SDA 或 SCL 被拉低），执行以下恢复步骤：

```
1. 禁用 I2C 模块
2. 配置引脚为 GPIO 输出
3. 发送 9 个时钟脉冲，让从机释放 SDA
4. 发送 STOP 信号
5. 重新配置 I2C
```

### 9.2 总线忙检测

QMI8658 正常工作需要 SDA=1 且 SCL=1。检测到总线异常时：

- **SDA=0, SCL=1**：可能是从机拉低总线，执行恢复
- **SCL=0**：可能是时钟 stretching，执行恢复
- **SDA=1, SCL=1**：总线空闲，可以开始通信

## 10. 常见问题排查

| 问题                  | 可能原因                    | 解决方案                          |
|----------------------|----------------------------|----------------------------------|
| WHO_AM_I 读取失败    | I2C 总线异常，地址错误     | 检查总线恢复，SDA是否被拉低      |
| RESET_STATE=0x00    | 软复位未生效，I2C写入失败  | 检查I2C写入是否收到ACK          |
| 数据全0xFF           | 芯片未就绪，读取过早       | 等待STATUS0的ready标志           |
| 数据跳变剧烈         | ODR过高或量程不合适        | 降低ODR或调整量程                |
| 陀螺仪数据不准       | 量程不合适                 | 根据实际运动范围选择合适量程     |
| 初始化超时           | 总线持续被拉低             | 检查QMC6309等其他I2C设备是否冲突  |
